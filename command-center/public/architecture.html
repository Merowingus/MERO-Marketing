Critical Rules — Agents Must Not Violate
1
Multi-tenant from Day 1
Every DB table has tenant_id + product_id. Isolation via Supabase RLS — never app-code conditions.
2
Design Tokens — Import, Never Re-declare
Tokens from MEROWINGUS Studio/website/Design/tokens. No hardcoded colors. Current local tokens in dashboard/styles.css = known tech debt; don't worsen it.
3
Read-First — Actions Are Phase 4+
Command center shows state now. Actions/MCP is designed now, implemented only in Phase 4. Do not implement ahead of schedule.
4
Platform Base — Not Tool Internals
Shared layer (tokens/auth/engine/MCP) lives in Studio. MERO Marketing is the first consumer, not the hub. Extract packages only when the second tool arrives.
5
Parallel Siblings — No Cross-Repo Coupling
MERO tools are separate repos, independent deploys. Cross-product work via MERO Product (Planner). Never import one tool's internals into another.
6
Zero Fabrication
Marketing content = what the product actually does. No invented results, fake metrics, or claims it can't back.
7
One Wheel at a Time
Capacity ~5–10h/week. Check ROADMAP.md before starting. Build and ship one agent or feature at a time.
8
Auth: getUser() not getSession()
Server-side auth must use supabase.auth.getUser() — validates token server-side. getSession() trusts client data — never use for security checks.
